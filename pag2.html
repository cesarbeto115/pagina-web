Ejemplos detallados

DDL – Definición de Datos

CREATE TABLE empleados (
            id INT PRIMARY KEY,
            nombre TEXT,
            edad INT
            );

Este comando crea una nueva tabla llamada empleados con tres columnas:
id: un número entero que funciona como clave primaria (no se repite).
nombre: un texto para el nombre del empleado.
edad: un número entero para la edad del empleado.



DML – Manipulación de Datos


            -- Insertar
            INSERT INTO empleados (id, nombre, edad) VALUES (1, 'Ana', 30);

            -- Consultar
            SELECT * FROM empleados;

            -- Actualizar
            UPDATE empleados SET edad = 31 WHERE id = 1;

            -- Eliminar
            DELETE FROM empleados WHERE id = 1;

-- Insertar --
Agrega un nuevo registro a la tabla empleados con:
id: 1, nombre: Ana, edad: 30

-- Consultar --
Consulta todos los registros de la tabla empleados . El * significa que traerá todas las columnas.

-- Actualizar --
Modifica el valor de la columna edad a 31 para el empleado cuyo id es 1.

-- Eliminar --
Elimina el registro del empleado cuyo id es 1.



DCL – Control de Acceso

       GRANT SELECT, INSERT ON empleados TO usuario1;
            REVOKE INSERT ON empleados FROM usuario1;

Da permisos al usuario llamado usuario1 para que pueda consultar (SELECT) e insertar (INSERT) datos en la tabla empleados.
Revoca (quita) al usuario usuario1 el permiso de insertar registros en la tabla empleados.



TCL – Control de Transacciones

BEGIN;

                UPDATE empleados SET edad = 35 WHERE nombre = 'Luis';

                -- Si todo está bien:
                COMMIT;

                -- Si algo sale mal:
                //ROLLBACK;
  • BEGIN inicia una transacción.
  • Luego actualiza la edad del empleado llamado Luis a 35.
  • Si no ocurre ningún error, COMMIT guarda permanentemente los cambios.
  • Si algo sale mal, ROLLBACK revierte todos los cambios hechos desde BEGIN.

    • Esto es útil para mantener la integridad de los datos, especialmente si hay varias operaciones que deben completarse juntas.
